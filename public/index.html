AI Agent

How the land-records agent works

An autonomous pipeline that takes raw client documents and returns verified land owner data, ready for a lawyer to draft against.

  1. 1

    Lawyer uploads documents

    Sale deeds, FIRs, identity proofs, or any paperwork the client has on hand. PDFs and images are both accepted.

  2. 2

    Data extracted from documents

    The agent reads each document and pulls structured fields — names, khasra numbers, village, tehsil, district, dates — with citations back to the source page.

  3. 3

    Data entered into the government website

    The agent logs into the relevant land-records portal and fills in the extracted fields, navigating dropdowns and search forms automatically.

  4. 4

    Output from the website captured

    Once results load, the agent captures the response page — including any Jamabandi records, mutation entries, or registry hits.

  5. 5

    Data extracted from the government output

    The captured page is parsed back into clean, structured rows — owner names, shares, parent records, and dates — normalized for downstream drafting.

  6. 6

    Available on the portal for lawyers to draft

    Verified land-owner data lands on the matter in Counsel, where the lawyer can drop it directly into a draft using the Draft Playground or template tools.
